# MandalartSync プレゼン構成案

## 想定尺

- 全体 3 分
- スライド 7 枚 + デモ

## ストーリーライン

1. 目標管理アプリは多いが、「やったつもり」を正せない
2. `MandalartSync` は目標と実行記録をつなぎ、認知のズレを可視化する
3. GitHub / Calendar / 自己申告を束ねて、刺さるフィードバックを返す
4. ハッカソン期間中でも回る SwiftUI ネイティブ実装として仕上げた

## スライド構成

### 1. タイトル

- キーメッセージ: 目標管理ではなく、自己認識を補正するアプリ
- 話す内容:
  - アプリ名 `MandalartSync`
  - チーム名
  - 一言で「認知のズレを突きつけるマンダラート」
- 視覚要素:
  - ホーム画面のスクリーンショット
  - アプリ名ロゴまたはタイトル文字
- 時間目安: 10 秒

### 2. 問題提起

- キーメッセージ: 目標は立てても、実際には「やった気」だけが残りやすい
- 話す内容:
  - 既存の目標管理アプリは記録が自己申告に寄りがち
  - 頑張った感覚と、残っている証拠がズレる
  - ハッカソンのような短期勝負ほど、このズレが致命傷になる
- 視覚要素:
  - 「やったつもり」と「実際の行動」の対比図
  - 薄いコミットログと強気な自己評価の対比
- 時間目安: 20 秒

### 3. ソリューション

- キーメッセージ: 目標、行動、外部ログをまとめて見てズレを検出する
- 話す内容:
  - マンダラートで目標を 32 項目に分解
  - カード UI でその日の行動をチェックイン
  - GitHub / Calendar と突き合わせて、客観ログと比較
- 視覚要素:
  - 目標 -> 行動 -> 証拠 -> フィードバック の流れ図
  - マンダラート画面と結果画面の 2 枚並び
- 時間目安: 25 秒

### 4. 主要機能

- キーメッセージ: 記録するだけでなく、見返したときに行動へ戻せる
- 話す内容:
  - 目標編集
  - デイリーチェックイン
  - アクションジャーナル
  - 認知のズレ分析と通知
- 視覚要素:
  - 4 画面のコラージュ
  - それぞれに短いラベルを付与
- 時間目安: 25 秒

### 5. 刺さる 1 シーン

- キーメッセージ: 「やったと思っていたのに、証拠がない」をその場で突きつける
- 話す内容:
  - `Day10` で作成したデモデータを適用
  - ホームの警告バナー -> ジャーナル -> 結果画面の順に見せる
  - 認知のズレスコアと次の一手まで一気に見せる
- 視覚要素:
  - 結果画面の警告カードを大きく配置
  - デモの 3 ステップを横並びで表示
- 時間目安: 20 秒 + デモ 40 秒

### 6. 技術的な挑戦

- キーメッセージ: UI モックで終わらせず、データが流れる体験まで SwiftUI で通した
- 話す内容:
  - `SwiftUI` で画面全体を構築
  - `SwiftData` でローカル保存
  - GitHub / Google Calendar の同期基盤
  - 認知のズレ分析、通知、強い警告演出
- 視覚要素:
  - 技術スタック図
  - データの流れを示す簡易アーキテクチャ図
- 時間目安: 20 秒

### 7. まとめ

- キーメッセージ: 目標達成を助けるのではなく、自己認識を正すことで前進を促す
- 話す内容:
  - 今日の自分の認識と、実際の行動を一致させる
  - 今後は Firebase 同期や実運用連携を強化
  - 「目標管理」から「自己認識補助」への転換を提案
- 視覚要素:
  - 一番強い画面の再掲
  - 今後の展望を 2 つ程度
- 時間目安: 20 秒

## デモ接続メモ

- 設定画面で `刺さる1シーン` を適用してから開始する
- 見せる順番は `ホーム -> 記録 -> 結果`
- API エラーを見せる場合は `APIエラー再現` を適用し、ホーム右上の同期ボタンを押す

## 使うスクリーンショット候補

- ホーム画面の警告バナー
- デイリーチェックイン画面
- アクションジャーナル
- 結果画面の認知のズレ分析
- 設定画面のデモ運用セクション

## 審査員向けに強調する点

- 新規性: 目標ではなく「認知のズレ」にフォーカスしている
- 技術力: UI だけでなく、ログ統合と分析導線まで実装している
- 完成度: デモデータと異常系まで準備済み
- プレゼン: 刺さる 1 シーンが明確で、短時間でも印象を残せる
